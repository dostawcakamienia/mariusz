🔐 Uprawnienia – chmod, chown, umask▶
Schemat uprawnień rwxrwxrwx
schemat
drwxr-xr-x  2  jan  pracownicy  4096  Jan 1  12:00  folder/
│└─────────┘     └─┘  └────────┘
│   rwxr-xr-x   właśc.  grupa
│
│ d = katalog | - = plik | l = link
│
│ rwx = właściciel (owner)
│  r-x = grupa (group)
│   r-x = pozostali (others)
│
│ r = odczyt (4)
│ w = zapis  (2)
│ x = wykonanie (1)
│
│ Przykład: chmod 755 = rwxr-xr-x
│           7 = 4+2+1 (rwx)
│           5 = 4+0+1 (r-x)
│           5 = 4+0+1 (r-x)

🚨 Obowiązkowo ▼

Przez GUI (Panel sterowania):

  1. Otwórz Centrum sieci i udostępniania

    Prawy klik ikona sieci → Otwórz ustawienia sieci i Internetu → Centrum sieci i udostępniania

  2. Kliknij połączenie sieciowe

  3. Właściwości → IPv4

    Właściwości → zaznacz Protokół IPv4 → Właściwości

  4. Wpisz dane

    Użyj następującego adresu IP, wpisz: IP, maskę, bramę, DNS

  5. OK i weryfikacja

    OK na wszystkich oknach, następnie ipconfig /all w CMD

# Weryfikacja

Quota – limity dyskowe użytkowników i grup

🚨 Obowiązkowo ▼
Quota działa na poziomie systemu plików (ext4/xfs). Samba automatycznie korzysta z systemowej quota Linux przy dostępie do udziałów.
  1. Instalacja pakietu quota
    bash
    sudo apt update && sudo apt install quota quotatool -y
  2. Edycja /etc/fstab – dodaj opcje quota
    bash
    sudo cp /etc/fstab /etc/fstab.bak
sudo nano /etc/fstab
    /etc/fstab – przykładowy wpis
    # PRZED zmianą (oryginalny wpis):
/dev/sda1  /  ext4  errors=remount-ro  0  1

# PO zmianie – dodaj usrquota,grpquota:
/dev/sda1  /  ext4  errors=remount-ro,usrquota,grpquota  0  1

# Lub dla osobnej partycji:
/dev/sdb1  /srv/samba  ext4  defaults,usrquota,grpquota  0  2
  3. Remontuj system plików
    bash
    sudo mount -o remount /
# Sprawdź czy opcje są aktywne
cat /proc/mounts | grep quota
# Powinno pokazać: usrquota i grpquota w opcjach
  4. Utwórz pliki bazy danych quota
    bash
    # -c=create, -u=user, -g=group, -m=force (nie odmontowuj)
sudo quotacheck -cugm /

# Sprawdź czy pliki quota powstały
ls -la /aquota.user /aquota.group
# Powinny istnieć oba pliki
  5. Włącz quota
    bash
    sudo quotaon -v /
# Komunikat: /dev/sda1 [/]: group quotas turned on
# Sprawdź status:
sudo quotaon -ps /
  6. Ustaw limity dla użytkownika
    bash
    sudo edquota -u jan
# W edytorze (nano/vi) zobaczysz tabelę:

# Filesystem  blocks  soft    hard  inodes  soft  hard
# /dev/sda1   100     0       0     1       0     0
#
# blocks = aktualnie zajęte (w KB)
# soft = miękki limit (ostrzeżenie)
# hard = twardy limit (blokada)

# Ustaw: soft=100MB, hard=150MB
# /dev/sda1  100  102400  153600  1  0  0

sudo edquota -g sambashare  # limit dla grupy
  7. Ustaw okres karencji (grace period)
    bash
    sudo edquota -t
# Ustaw grace period: np. "7 days" dla bloków
  8. Skopiuj limity do innych użytkowników
    bash
    # Skopiuj limity z jan do innych użytkowników
sudo edquota -p jan anna
sudo edquota -p jan piotr
# Kopiowanie limitów z użytkownika wzorcowego na innego użytkownika\nsudo edquota -p uzytkownik_wzorcowy nowy_uzytkownik')">Kopiuj
    sudo edquota -p uzytkownik_wzorcowy nowy_uzytkownik
# Przenosi konfigurację miękkiego i twardego limitu na nowego użytkownika
  • Zweryfikuj i wygeneruj raport limitów
    bash
    sudo repquota -avug
# Wyświetla pełny raport zajętości dysku oraz nałożonych limitów quota

    Scenariusz Egzaminacyjny: Zintegrowany Serwer Usług (Linux + Windows)

    Treść zadania: Skonfiguruj serwer Linux jako router dla sieci lokalnej. Serwer posiada dwa interfejsy: eth0 (zewnętrzny) oraz eth1 (sieć lokalna). Stacja robocza Windows ma otrzymać adres IP automatycznie. Na Linuxie uruchom serwer WWW oraz serwer FTP zabezpieczony przed użytkownikami anonimowymi.

    Wymagania szczegółowe:

    1. Adresacja interfejsu lokalnego eth1: 192.168.10.1/24
    2. Serwer DHCP ma przydzielać adresy z zakresu 192.168.10.100 - 192.168.10.150
    3. Serwer FTP ma zezwalać na logowanie użytkownikowi lokalnemu egzaminator, blokując dostęp anonimowy i zamykając go w jego katalogu domowym (chroot).

    Krok 1: Adresacja sieciowa na Linux (/etc/network/interfaces):

    auto eth1
iface eth1 inet static
    address 192.168.10.1
    netmask 255.255.255.0

    subnet 192.168.10.0 netmask 255.255.255.0 {
    range 192.168.10.100 192.168.10.150;
    option routers 192.168.10.1;
    option domain-name-servers 192.168.10.1;
}

    Krok 3: Konfiguracja zabezpieczeń serwera FTP (/etc/vsftpd.conf):

    anonymous_enable=NO
local_enable=YES
write_enable=YES
chroot_local_user=YES
allow_writeable_chroot=YES

    C:\> ipconfig /renew
C:\> ping 192.168.10.1
C:\> ftp 192.168.10.1

    📝 Procedura wykonania kabla sieciowego▶
    • Zdjęcie otuliny zewnętrznej

      Wsuń koniec kabla do ściągacza izolacji (strippera) na odległość ok. 2.5 – 3 cm. Obróć narzędzie wokół kabla, zdejmij naciętą osłonę, uważając, aby nie uszkodzić wewnętrznych żył.

    • Rozplecenie i wyprostowanie par przewodów

      Rozpleć cztery pary przewodów. Każdą żyłę dokładnie wyprostuj palcami, likwidując wszelkie zagięcia struktury miedzianej (ułatwi to równe ułożenie i wsunięcie żył do wtyku).

    • Ułożenie sekwencji barwnej

      Ułóż przewody płasko, obok siebie, ściśle według wybranego standardu egzaminacyjnego (np. od lewej do prawej: biało-pomarańczowy do brązowy dla T568B). Mocno ściśnij ułożoną taśmę przewodów palcami.

    • Równe przycięcie żył kabla

      Trzymając pewnie ułożone żyły, odetnij ich nadmiar pod kątem prostym za pomocą ostrza obcinarki. Pozostaw żyły o długości dokładnie ok. 1.2 – 1.4 cm. Zbyt długie żyły uniemożliwią dociśnięcie zewnętrznej otuliny kabla przez klin wtyku; zbyt krótkie nie dosięgną blaszek stykowych.

    • Wprowadzenie przewodów do wnętrza wtyku RJ-45

      Trzymaj przezroczysty wtyk RJ-45 plastikowym zatrzaskiem (klipsem) skierowanym do dołu (miedziane piny patrzą na Ciebie). Wsuń przygotowaną taśmę żył zdecydowanym ruchem do samego końca, tak aby każda żyła weszła w swój kanalik prowadzący.

    • Kontrola docisku czołowego oraz otuliny

      Spójrz na czoło wtyku – miedziane końce wszystkich 8 żył muszą ściśle dotykać przedniej, przezroczystej ścianki. Upewnij się, że gruba, zewnętrzna powłoka kabla weszła głęboko do wnętrza wtyku pod plastikowy klin zaciskowy.

    • Zaciśnięcie złącza w zaciskarce i weryfikacja testerem

      Włóż wtyk do komory 8P zaciskarki i mocno ściśnij rączki narzędzia do końca. Powtórz całą procedurę dla drugiej strony kabla. Następnie podłącz oba końce do testera okablowania – dla kabla prostego diody od 1 do 8 (oraz G, jeśli kabel jest ekranowany STP) muszą zapalać się synchronicznie na obu modułach.

    2. Montaż: Sprawdź pozycję wycięcia (klucza) na dolnej krawędzi kości pamięci i dopasuj ją do plastikowego wypustu wewnątrz slotu DIMM. Wsuń kość prostopadle i równomiernie dociśnij oburącz z obu stron, aż zatrzaski gniazda same zatrzasną się z wyraźnym kliknięciem.
    ⚡ Zasilacz (PSU) Diagnoza i Serwis

    🔍 Objawy awarii: Całkowity brak reakcji komputera na przycisk włącznika, wyłączanie jednostki pod obciążeniem, niestabilne napięcia na liniach pomiarowych multimetru.

    🛠️ Procedura serwisowa krok po kroku
    1. Demontaż: Odepnij wszystkie przewody zasilacza z wnętrza komputera: ATX 24-pin (płyta główna), EPS 4/8-pin (zasilanie CPU), PCI-E (karta graficzna), złącza SATA/Molex (dyski i wentylatory). Odkręć 4 śruby mocujące z tyłu obudowy i ostrożnie wysuń zasilacz.
    2. Montaż: Umieść nowy zasilacz w zatoce obudowy (wentylatorem do dołu, jeśli obudowa ma otwór wentylacyjny na dole). Przykręć go do panelu tylnego obudowy 4 śrubami. Podłącz wszystkie wtyczki zasilania komponentów, upewniając się, że plastikowe zatrzaski stabilnie zablokowały się na gniazdach.
    🎮 Karta Graficzna (GPU) Diagnoza i Serwis

    🔍 Objawy awarii: Artefakty na monitorze (linie, geometryczne zniekształcenia), czarny ekran pomimo poprawnego rozruchu dysków, BIOS generuje 1 długi i 2 krótkie sygnały.

    🛠️ Procedura serwisowa krok po kroku
    1. Demontaż: Odłącz kable wideo (HDMI/DisplayPort) od tylnego panelu GPU. Odłącz wewnętrzne przewody zasilające PCI-E (6/8-pin). Odkręć śruby mocujące śledź karty do obudowy. Ważne: naciśnij plastikową dźwignię blokującą na końcu gniazda PCI-Express x16 na płycie głównej, a dopiero wtedy wyjmij kartę pionowo ze slotu.
    2. Montaż: Wyrównaj złącze krawędziowe nowej karty ze slotem PCI-Express x16 i dociśnij ją pionowo w dół, aż plastikowa blokada slotu automatycznie odskoczy i zablokuje kartę. Przykręć metalowy śledź karty śrubami do obudowy i podłącz dodatkowe przewody zasilające PCI-E.
    🔥 Procesor (CPU) + Chłodzenie Diagnoza i Serwis

    🔍 Objawy awarii: Natychmiastowe wyłączenie PC po kilku sekundach od startu (zabezpieczenie termiczne przed przegrzaniem), pętla restartów (bootloop), płyta główna nie przechodzi testu CPU.

    🛠️ Procedura serwisowa krok po kroku
    1. Demontaż: Odłącz przewód wentylatora z gniazda CPU_FAN. Odkręć śruby chłodzenia w układzie krzyżowym (X). Delikatnie przekręć radiator lekko w lewo i prawo, aby zerwać wiązanie zaschniętej pasty (nie ciągnij mocno w górę!). Zdejmij chłodzenie. Podnieś metalową dźwignię gniazda procesora (socketu) i chwytając CPU za brzegi, wyjmij go pionowo.
    2. Montaż: Zlokalizuj trójkątny znacznik (klucz) w rogu nowego procesora oraz na obudowie gniazda. Włóż procesor delikatnie i idealnie prostopadle do gniazda (bez naciskania). Opuść i zablokuj metalową dźwignię socketu. Oczyść powierzchnie alkoholem izopropylowym (IPA), nałóż na środek procesora świeżą pastę termoprzewodzącą (wielkość ziarna grochu), zamontuj radiator dokręcając śruby na krzyż i wepnij kabel wentylatora do gniazda CPU_FAN.
    💾 Dyski Twarde i SSD (SATA / M.2) Diagnoza i Serwis

    🔍 Objawy awarii: Komunikat No Boot Device Found, drastyczny spadek wydajności i zawieszanie systemu operacyjnego, stukanie głowicy (HDD), błędy S.M.A.R.T. (bad sektory).

    🛠️ Procedura serwisowa krok po kroku
    • Dysk SATA (3.5" HDD / 2.5" SSD):
      • Wymiana: Odepnij kabel sygnałowy SATA oraz kabel zasilający SATA (zwróć uwagę na metalowe zatrzaski wtyczek). Odkręć śruby mocujące dysk z boku zatoki obudowy. Zamontuj nowy dysk, zabezpiecz śrubami i podłącz przewody (kabel sygnałowy wepnij do portu SATA_0 lub SATA_1 na płycie głównej).
    • Dysk M.2 (SATA / NVMe PCIe):
      • Wymiana: Odkręć śrubkę zabezpieczającą dysk na końcu slotu M.2 (lub zwolnij wbudowany radiator/zatrzask). Dysk lekko uniesie się pod kątem 30 stopni – wysuń go delikatnie z gniazda. Nowy dysk wsuń w slot pod tym samym kątem, dociśnij lekko do laminatu płyty i przykręć śrubką montażową M.2.

    🔥 TOP 3 Najtrudniejsze zadania praktyczne CKE
    1. Windows GPO: Zablokowanie wiersza poleceń (CMD)▶
    1. Utwórz i przypisz GPO

      Menedżer serwera → Narzędzia → Zarządzanie zasadami grupy. Kliknij PPM na docelowe OU (np. "Uczniowie") → Utwórz obiekt GPO w tej domenie i zakorzeń go tutaj.

    2. Edytuj polisę

      PPM na nowy obiekt → Edytuj. Przejdź do: Konfiguracja użytkownika → Zasady → Szablony administracyjne → System.

    3. Włącz blokadę

      Znajdź zasadę Zapobiegaj dostępowi do wiersza polecenia. Kliknij dwukrotnie, ustaw na Włączone i zatwierdź.

    4. Weryfikacja

      Zaloguj się na klienta jako dany użytkownik, wciśnij Win+R, wpisz cmd. Zobaczysz komunikat o blokadzie nałożonej przez administratora.

    2. Linux Quota: Nałożenie twardego limitu 200MB dla użytkownika▶
    1. Otwórz edytor limitów
      bash
      sudo edquota -u uczen
    2. Zmień wartość "hard"

      Znajdź kolumnę hard (domyślnie 0). 1 blok = 1 KB. Aby ustawić 200 MB, zmień 0 na 200000. Zapisz plik (Ctrl+O, Ctrl+X w nano).

    3. Weryfikacja
      bash
      sudo repquota -a   # Sprawdza czy limit dla użytkownika jest aktywny
    3. Linux SSH: Logowanie za pomocą klucza (bez hasła)▶
    1. Generowanie pary kluczy na kliencie
      bash (klient)
      ssh-keygen -t rsa   # Wciskaj 3x Enter, by zostawić wartości domyślne (bez hasła na klucz)
    2. Przesłanie klucza na serwer
      bash (klient)
      ssh-copy-id uczen@192.168.1.10   # Będziesz musiał podać hasło użytkownika 'uczen' ten ostatni raz
    3. Weryfikacja

      Zaloguj się ponownie poleceniem ssh uczen@192.168.1.10 – powłoka połączy Cię od razu, bez pytania o hasło tekstowe.

    🚑 DIAGNOSTYKA Usługa nie działa - co sprawdzić?
    1. Usługi sieciowe i WWW (Apache, DHCP, DNS)▶

    🌐 HTTP Apache: 403 Forbidden? Złe uprawnienia do folderu (chown -R www-data:www-data /var/www) albo domyślna strona blokuje nowy vhost. Użyj a2dissite 000-default.conf.

    📡 DHCP: Klient dostaje IP z puli APIPA (169.254.x.x)? Błąd składni w dhcpd.conf (brak średnika) lub nie podałeś nazwy karty sieciowej w pliku /etc/default/isc-dhcp-server.

    🔍 DNS Bind9: Server failed przy nslookup? Prawdopodobnie zapomniałeś kropki na końcu domen w pliku strefy lub pomyliłeś odwrócony zapis adresów w strefie reverse. Użyj named-checkzone.

    2. Transfer plików i autoryzacja (FTP, Samba, SSH, AD)▶

    📁 VSFTPD: Błąd "530 Login incorrect" pomimo dobrych danych? Usługa gryzie się z chrootem. Koniecznie dopisz parametr allow_writeable_chroot=YES do pliku konfiguracyjnego.

    📂 SAMBA: Windows odrzuca hasło, mimo że konto w Linuksie istnieje? Samba używa własnej bazy poświadczeń sieciowych. Pamiętaj o użyciu smbpasswd -a uczen.

    🔐 SSH: Connection refused? Upewnij się, że nie zmieniłeś portu w sshd_config, a jeśli tak, to musisz wskazać go u klienta (ssh -p 2222 uczen@IP).

    🖥️ Active Directory (WIN): Nie możesz zalogować stacji klienckiej na usera z domeny? Pierwsza zasada – w ustawieniach IPv4 klienta, serwer DNS musi bezwzględnie wskazywać na adres IP kontrolera domeny AD.
